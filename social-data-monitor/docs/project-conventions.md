# 项目约定

## 授权业务数据保持完整

本项目在授权访问的业务功能中优先保留完整原始数据，避免因为过度脱敏影响监控、排查和分析效果。

- 数据库内的采集结果、API 原始响应、审计记录和导出数据，应按业务需要保存完整值。
- 前端页面在具备授权访问控制的前提下，可以展示完整业务数据。
- 静态占位、摘要、截断或哈希不应替代真实业务字段，除非产品需求明确要求。
- 静态存储加密、认证、授权、软删除和存储边界属于安全控制手段，不等同于业务数据脱敏。

## 公开内容必须脱敏

公开仓库、公开 issue、Pull Request、截图、文档示例、演示日志和提交历史不得包含真实密钥或个人隐私数据。

- 不提交真实 Cookie、token、refresh token、数据库密码、私钥、浏览器 profile 或个人账号凭据。
- `.env.local`、运行数据、日志、外部研究 dump 和本地工具缓存只保留在本地。
- 文档示例使用 `<your_db_password>`、`SESSDATA=...` 等占位符。
- 需要定位问题时，先在本地保留完整排查材料；对外沟通前再做必要脱敏。
